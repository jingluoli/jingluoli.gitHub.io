线程安全

指多线程环境下，程序或系统能够正确处理多个线程同时访问共享资源的情况，而不会导致数据不一致或其他不可预期的行为。

相关概念

1.竟态条件：当多个线程访问共享数据并试图同时修改它时，最终结果取决于线程执行的顺序
2.临界区：访问共享资源的代码段，需要同步机制保护
3.原子性：操作不可分割，要么完全执行，要么完全不执行

实现线程安全的方法

1. 互斥量(Mutex)
最基本的同步机制，用于保护共享数据

2. 条件变量
用于线程间的通信，允许线程等待特定条件成立


3. 原子操作
定义：指在多线程或并发编程中不可分割的操作，这些操作要么完全执行，要么完全不执行，不会被其他线程的操作打断。
对简单的数据类型，可以使用原子操作避免锁的开销


4. 读写锁
C++14引入共享互斥量，允许多个读线程同时访问


5. 信号量
C++20引入，vs编译需要将C++标准切换到C++20版本


6. 屏障（Barrier）
C++20引入了屏障，用于同步多个线程在某个点汇合

windows thread、std::thread、Qthread

windwos thread

windows API，操作系统级别的线程，提供最底层的线程控制(如优先级、安全属性等)
1. 创建方式：
CreateThread()

2. 启动方式：直接运行
3. 线程同步
waitForSingleObject、临界区、事件
4. 线程通信
共享内存、消息队列
5. 生命周期
手动管理 closeHandle()
6. 性能与开销
最底层，性能最优，但管理复杂

std::thread

C++11标准库，跨平台的轻量级线程封装，基于RAII管理线程生命周期
1. 构造函数

2. 启动方式：start()
3. 线程同步
std::mutex、std::condition_variable
4. 线程通信
共享变量、std::future
5. 生命周期
join()或detach()
6. 性能与开销
轻量级、跨平台、适合通用多线程任务

QTHread

Qt框架，高级线程管理器，与Qt信号槽机制深度集成，支持事件循环
1.创建方式：
继承或moveToThread


2. 启动方式： start()
3. 线程同步
QMutex、QWaitCondition
4. 线程通信
信号槽（跨线程安全）
5. 自动清理
deleteLater
6.因事件循环和信号槽机制，有一定开销，但开发效率高

线程间通信方式

共享内存

直接读写进程内的共享变量或数据结构（如全局变量、堆内存）
问题：需要通过同步机制避免竞态条件

锁机制

1. 互斥锁(mutex)：保证同一时间仅一个线程访问共享资源
2. 读写锁：允许多读单写
3. 自旋锁：忙等待，适用于短时间阻塞

条件变量

线程等待某个条件成立，另一个线程触发条件后唤醒等待者。常与互斥锁配合使用

信号量

计数器控制资源访问（如限制并发线程数）
二元信号量可模拟互斥锁

屏障（Barrier）

同步多个线程的执行阶段，所有线程到达屏障点后才继续

消息队列

生产者-消费者模型，通过队列传递消息（如C++de std：：queue+锁）

进程间通信

进程拥有独立地址的空间，通信需通过内核或共享存储介质，开销较大

管道 Pipe

1. 匿名管道：单向通信，父子进程间使用
2. 有名管道：通过文件系统实现，无关进程可通信

消息队列（Message Queue）

内核维护的链表结构，进程通过标识符读写消息，支持优先级、异步通信

共享内存（shared Memory）

映射同一块物理内存到多个进程的地址空间，速度最快，需配合信号量等同步机制

信号（Signal）

内核通知进程异步事件，但信息量较小，可用于简单通知，非结构化数据

套接字(Socket)

跨网络或本机通信（如TCP/UDP套接字），灵活性高，但开销较大

文件/内存映射文件（Memory-Mapped File）

文件作为共享介质，或通过mmap将文件映射到内存

RPC(Remote Procedure Call)

跨进程/网络的函数调用抽象（如gRPC）

死锁

多线程/多进程系统中一种常见的资源竞争问题，指两个或多个线程/进程因相互等待对方持有的资源而无线阻塞的状态。死锁会导程序挂起，无法继续执行。

死锁产生的必要条件

1. 互斥条件：资源一次只能由一个线程或进程占用（锁，文件，打印机等）
2. 占有并等待：线程/进程持有至少一个资源，并等待获取其他被占用的资源
3. 非抢占条件：已经分配的资源不能被强制剥夺，必须由持有者主动释放
4. 循环等待：存在一个循环等待链，如：线程A 持有资源1，等待资源2；线程B 持有资源2，等待资源1。

死锁的解决方案

1. 预防死锁
·破坏互斥条件：某些资源可共享（如只读文件），但多数资源（如锁）必须互斥
·破坏占有并等待：要求线程一次性申请所有所需资源
·破坏非抢占条件：允许抢占资源，但实现复杂
·破坏循环等待：规定资源的线性申请顺序
2. 避免死锁（运行时检测）
·银行家算法：系统动态检查资源分配是否会导致死锁，仅允许安全状态下的请求
·锁层次：定义锁的获取顺序，违反顺序则报错
3. 检测与恢复
·死锁检测：通过资源分配图（Resource Allocation Graph, RAG）检测环路。操作系统定期扫描进程等待关系。
·恢复措施：终止进程（如强制结束一个死锁进程）。回滚（Rollback）到安全状态。
